Release Note:

  • 2015-04-22 : Bug reported and fixed. Store Notes null checking is added before proceeding with upload. Also, more information and exception tracings are added.

  • 2015-04-10 : Bug reported and fixed. Historical priority questions and future ones are sent down to MyDay, according to Promotion Configurations.

  • 2015-04-10 : Bug reported and fixed. If a local/regional chain is selected, the parents chain id is sent down to MyDay.

  • 2015-04-09 : Bug found and fixed. Both Reporting and Servicing territory is used for map filtering. During deployment to INT the End Date fix is overriden. Redeployed again.

  • 2015-04-08 : Bug fixed, if Region/Division/Zone is selected in bottler hierarchy, then SystemID is ignored.

  • 2015-04-08 : One day Question bug fix, the Questions are sending down to MyDay from StartDate ~ EndDate, including both start and end date.

  • 2015-04-06 : ForAllChains bug fixed.

  • 2015-03-30 : DB changes deployed to INT for integration test and web service is pointed to BSCCSQ07/Portal_Data_INT.

  • 2015-03-25 : Empty Image bug is found and resolved. If any image is empty, then it won't get saved to SharePoint.

  • 2015-03-24 : IsFairShare updated to be integer field and is interpreted with DB Table BCMyDay.TieInFairShareStatus, field TieInFairShareStatusID.

  • 2015-03-23 : IsFairShare field added to the StoreDisplay this change is propagated to DB and echoed back to TieInHistory.

  • 2015-03-23 : DisplayLocationID in StoreTieInUpload input message is updated to be ClientDisplayID and this change is propagated to DB and echoed back to TieInHistory.

  • 2015-03-23 : Method GetStoreTieInHistory is removed.

  • 2015-03-23 : Promotion States relationship table is removed from the output for GetPriorityQuestionsByRegionID.

  • 2015-03-23 : Method GetManagementPriorityIDsByBottler removed. Table BCMyday.PriorityBottlerForUI dropped.

  • 2015-03-20 : Method GetManagementPriorityIDsByBottler added.

  • 2015-03-20 : Promotion States relationship table is added to the output for GetPriorityQuestionsByRegionID.

  • 2015-03-20 : Promotion States relationship table is added to the output for GetPromotionsByRegionID. Also, for service hosted at v-portals5, the db points to vh-portal2/Portal_Data204. Services hosted at webservices-int/myday-int points to BSCCAP108.

  • 2015-03-17 : NoteID, PromotionExecutionID and PriorityExecutionID are put into output for history. DisplayID in PromotionExecution accepts 0 or -1. Both are interpreted as null.

  • 2015-03-16 : InformationCategory field is passing the raw value of 'Promotion' or 'Non-promotion'.

  • 2015-03-16 : ClientStoreNoteID, ClientPromotionExecutionID and ClientPriorityExecutionID are updated to be string fields.

  • 2015-03-12 : Upload store display details db access code is optimized to save a db commit.

  • 2015-03-12 : Bug fixed, the connection string is not consistent, so the following error was reported "The INSERT statement conflicted with the FOREIGN KEY constraint \"FK_StoreConditionDisplay_StoreCondition\". The reason is that the StoreCoditon and its displays were being saved to different Databases.

  • 2015-03-11 : ClientPromotionExecutionID and ClientPriorityExcecutionID added. PromotionExecutions and PriorityExecutions added as well.

  • 2015-03-10 : This service point to Portal_Data on server BSCCAP108.

  • 2015-03-09 : InformationCategory field in Promotion is populated with "0" or "1". The field is from table Playbook.RetailPromotion. "Promotion" is translated to "1" and "Non-Promotion" to "0", same convention has been applied for Promotions for DSD route.

  • 2015-03-07 : Upload method rewritten so any combination of Displays/TieIns/Notes/Promotion Execution/Priority Execution with the StoreCondition will work. New implementation uses DBML(Old LINQ to SQL) so lots of SPs in use today in Prod are obsolete. For example, insert multiple objects passed in with XML encoded strings. Primary Key and identity column added to table StoreConditionDisplayDetail. Foreign reference with cascading delete added between StoreConditionDisplay and Detail table. Foreign references also added to all Children tables of StoreCondition. When same store condition comes again from MyDay, orphaned images get deleted as well. Also the ModifiedDate field in Priority is in right format of mm/dd/yyyy.

  • 2015-03-06 : Json interfaces locked in, after discussion with MyDay team. Everything works, just that Upload TieIn/Display still go through the old code path. This service point to Portal_Data204 on server Vh-Portal2. Chrome extension Advanced Rest Client is used for debugging. I don't know if this client supports User Agent, or any other client for that matter. So it's untested if a http request with User Agent(from MyDay) will actually get the GSN/User Name set. Also, this is an existing functionality so it's expected to be working as it is.
